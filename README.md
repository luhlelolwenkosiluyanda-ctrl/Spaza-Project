🛒NSIZWA TUCKSHOP INVENTORY & SALES SYSTEM
THEME: BUSINESS AND INFORMAL TRADE

MINI-PROJECT IDEA: Tuckshop Inventory and Sales summary Management.

PROJECT OVERVIEW(reviewed)
This project focuses on developing a simple software system to manage a tuckshop’s inventory and track daily sales. The system will allow the user to record items in stock, update quantities when products are purchased or restocked, and generate a summary of sales over a specific period.


TECH TOOLS:
Language - Java21+
Version Control - Gitbash & GitHub
Main Focus - Object-Oriented Programming (OOP)

KEY FEATURES:
Inventory Tracking: Real-time monitoring of stock levels (e.g., snacks, drinks).
Automated Sales Processing: Calculates totals and automatically reduces inventory upon successful sale.
Validation: Prevents sales if stock levels are insufficient.
Transaction History: Maintains a record of all processed sales for end-of-day summaries.

FILE STRUCTURE:
[Class---Responsibility]
Item.java---"Defines product attributes (Name, Price, Stock Level)."
Sale.java---Represents a single transaction and handles receipt formatting.
SaleService.java---"The core ""engine"" that processes sales and connects items to the inventory."
Inventory.java---Manages the collection of all items available in the shop.

HOW TO RUN:
1. Open the project in VS Code or other IDE.
2. Navigate to src/ub/com/csi142/tuckshop/products/SaleService.java (for example.)
3. Click Run above the main method.
4. Follow the prompts that follow.
